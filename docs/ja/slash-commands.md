# スラッシュコマンド

[English version](../en/slash-commands.md)

Typster は Zed の AI アシスタントパネル用に2つのスラッシュコマンドを提供します。Typst のリファレンスドキュメントをコンテキストとして挿入することで、アシスタントが Typst の構文と関数について正確に回答できるようにします。

## 使い方

Zed で AI アシスタントパネルを開き、`/` を入力すると利用可能なスラッシュコマンドが表示されます。Typster は以下を追加します：

- `/typst-docs`
- `/typst-symbols`

どちらのコマンドも引数のタブ補完に対応しています。

---

## `/typst-docs <関数名>`

Typst 標準ライブラリ関数のシグネチャ、パラメータ、使用例を表示します。

**例**:
```
/typst-docs heading
```

これにより `heading` のドキュメント — シグネチャ、すべてのパラメータ（型とデフォルト値付き）、使用例 — がアシスタントのコンテキストに挿入されます。

**タブ補完**: 関数名の一部を入力してTab を押すと候補が表示されます。例えば `tex` と入力して Tab を押すと `text` に補完されます。

**よく使う関数**: `heading`, `text`, `image`, `table`, `figure`, `cite`, `bibliography`, `math.equation`, `grid`, `box`, `block`, `list`, `enum`, `terms`, `link`, `ref`, `label`

---

## `/typst-symbols <カテゴリ>`

カテゴリ内のすべての数式シンボルを Typst 名と Unicode 文字で表示します。

**例**:
```
/typst-symbols arrow
```

**利用可能なカテゴリ**:

| カテゴリ | 説明 |
|---------|------|
| `arrow` | 矢印記号（→, ←, ↑, ↓, ⇒, ⟹, …） |
| `greek` | ギリシャ文字（α, β, γ, δ, π, σ, …） |
| `operator` | 数学演算子（∑, ∏, ∫, ∂, …） |
| `relation` | 関係記号（=, ≠, <, >, ≤, ≥, ≈, …） |
| `set` | 集合論記号（∈, ∉, ⊂, ⊃, ∪, ∩, …） |
| `logic` | 論理記号（∧, ∨, ¬, ⊕, ∀, ∃, …） |
| `accent` | アクセント記号（â, ë, î, ô, …） |
| `misc` | その他の記号（∞, ∅, ℝ, ℕ, …） |

**タブ補完**: カテゴリ名の一部を入力して Tab を押します。例えば `log` と入力すると `logic` に補完されます。

---

## なぜこれらのコマンドを使うのか？

これらのコマンドがない場合、AI アシスタントは Typst の関数名やシンボルコードを誤って推測することがあります。実際のドキュメントを挿入することで、アシスタントが以下について正確な情報を持つことができます：

- 関数のシグネチャと必須・任意パラメータ
- 数式モードで使用する正しいシンボル名
- 列挙型パラメータの有効な値
