# トラブルシューティング

[English version](../en/troubleshooting.md)

## tinymist が起動しない

**症状**: 補完なし、診断なし、ステータスバーに "Typst language support" が表示されない。

1. Zed のログを確認: **Zed > View > Logs**（またはコマンドパレットで `zed: open log`）。`tinymist` に関するエラーを探す。
2. カスタムの `binary.path` を設定している場合は、パスが正しく、バイナリが実行可能であることを確認する。
3. macOS Intel の場合: Typster が自動ダウンロードするバイナリは arm64 専用です。tinymist を手動インストールして `lsp.tinymist.binary.path` を設定してください。
4. ウィンドウをリロード: コマンドパレットで `zed: reload`。

## PDF が更新されない

**症状**: PDF ビューワーが古いバージョンのドキュメントを表示している。

- **外部ビューワーモード**: PDF は保存時に再生成されます（`exportPdf: "onSave"`）。ファイルを保存したことを確認してください。ビューワーが自動リロードしない場合は、ビューワーの自動リロード設定を確認してください。
- **ブラウザプレビューモード**: プレビューはリアルタイムで更新されるはずです。更新が止まっている場合は、ブラウザタブ（`http://127.0.0.1:23635`）をリロードしてください。

## ブラウザプレビューが読み込まれない

**症状**: `http://127.0.0.1:23635` で接続エラーが表示される。

1. tinymist が起動しているか確認（上記「tinymist が起動しない」を参照）。
2. tinymist 0.13.6 以降が使用されているか確認。ログで tinymist のバージョンを確認する。
3. ポート 23635 が他のプロセスで使用されていないか確認: `lsof -i :23635`
4. `typsterPreviewer: "browser"` を明示的に設定してブラウザモードを強制してみる。

## フォーマッターが動かない

**症状**: **Format Document**（⌥⇧F）が何もしない、またはエラーが表示される。

1. `typstyle` または `typstfmt` がインストールされ、PATH に通っているか確認:
   ```sh
   which typstyle   # または: which typstfmt
   ```
2. バイナリはインストールされているが検出されない場合は、インストール後に Zed を再起動してください（Typster は起動時に PATH を確認します）。
3. 設定でフォーマッターを明示的に指定することもできます:
   ```jsonc
   { "lsp": { "tinymist": { "settings": { "formatterMode": "typstyle" } } } }
   ```

## Forward Search が効かない

**症状**: Zed でカーソルを移動しても PDF ビューワーがスクロールしない。

- **ブラウザプレビュー**: Forward Search は自動動作します。効かない場合は、ブラウザタブが `http://127.0.0.1:23635` で開いていることを確認し、リロードしてください。
- **外部ビューワー**: Zed のログで検出されたビューワーを確認してください。間違ったビューワーが検出された場合（または検出されなかった場合）は、`typsterPreviewer` で明示的に指定してください。

## Inverse Search が効かない

ビューワーごとのセットアップ手順は [inverse-search.md](inverse-search.md) を参照してください。

**クイックチェックリスト**:
- `zed` CLI が PATH に通っているか? `which zed` を実行して確認。
- ビューワーの設定にエディタコマンドが保存されているか?
- `zed` CLI でファイルを開けるか? ターミナルで `zed /path/to/file.typ:10` を試してみる。

## セマンティックトークンのハイライトが正しくない

トークンの色がおかしい、または Typster なしの場合と同じに見える場合:

1. 設定で `semanticTokens` が `"disable"` に設定されていないか確認する。
2. テーマがセマンティックトークンのスコープに対応している必要があります。別の Zed テーマに切り替えてみてください。

## Zed の拡張機能ログを確認する方法

1. コマンドパレットを開く（**⌘⇧P**）
2. `zed: open log` を実行
3. `tinymist` または `typster` を含む行を探す

または、メニューから: **Zed > View > Logs**。
